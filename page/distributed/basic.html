术语

  1. 分布式

    分布式有两种不同的体现方式：分布式系统，分布式用户。一般开发时分布式指的是系统分布式部署。

  2. 集群

    多个服务器构成了一个群组，以单一系统的模式加以管理，用以提高服务的可用性和可缩放性。。

  3. 负载均衡 & 反向代理

    • 负载： web系统中一般指系统的用户承受能力。

    • 负载均衡 ：用多个服务器分担用户的访问，使多个服务器的负载达到均衡。可以使用硬件或软件实现。

    • 反向代理 ： 不考虑具体提供的的某一台服务器，而是通过外置的代理服务器进行统一访问。

  4. Session共享

    服务器是独立的，一般情况下，无法共享JVM数据，所以Session无法共享，必须采用第三方组件（Memcached, Redis）实现Session共享。

  5. 动静资源分离

    动态资源和静态资源都会向服务器发送请求，那么大量的静态资源请求会导致服务器效率变低，运行变慢，所以进行资源的动静分离。

集成

  1. 用户发送请求，不再直接发送给真实服务器，而是先发送到反向代理服务器，所以首先配置反向代理服务器Nginx，解压缩nginx-1.9.2.zip文件到无中文，无空格的目录中。

  2. 解压缩tomcat.zip文件到无中文，无空格的目录中，修改其中的端口号（8105, 8109, 8181）。

  3. 将解压后的tomcat，再复制一份，形成多个服务器。配置独立的端口号（8205, 8209, 8282）。

  4. 修改Nginx服务器的核心配置文件：conf/nginx.conf，用于将反向代理服务器和多个Tomcat连接在一起，形成服务器集群。

    解除L34注释，增加对应端口号。配置多服务器集群

    L48行，监听服务端口，当用户访问此端口时，进行反向代理服务

    修改L66, L70行，设定静态资源访问路径

  5. 安装Memcached缓存组件，用于实现服务器集群时的Session共享。解压缩memcached.zip文件，执行DOS命令进入到memcached解压缩后的目录，再执行DOS命令memcached.exe -d install进行安装，然后执行DOS命令memcached.exe -d start启动缓存服务。

测试

  1. 分别启动Nginx服务器和Tomcat服务器（N）。确认是否可以正常启动

  2. 正常启动后，分别在两个Tomcat服务器的webapps目录中添加ROOT文件夹并增加index.jsp页面

    在第一个Tomcat服务器的index.jsp增加内容

    Tomcat 1 INDEX JSP!

    在第二个Tomcat服务器的index.jsp增加内容

    Tomcat 2 INDEX JSP!

    启动所有服务器，浏览器访问80端口，刷新几次，观察页面内容变化，如果页面内容依次变化，那么说明反向代理和服务器集群配置成功。

  3. 在第二个Tomcat服务器中增加文件data.jsp，增加内容

    <% session.setAttribute("username", "zhangsan");>

    修改两个Tomcat服务器的index.jsp，尾部增加

     ${username}

    重新启动Tomcat服务器，浏览器访问80端口，刷新几次，观察页面内容变化，如果两个服务器的页面均出现zhangsan，那么说明Session共享配置成功。

  4. 在两个Tomcat服务器的index.jsp文件中增加内容

    <img src="pic.jpg">

    重新启动Tomcat服务器，浏览器访问80端口，刷新几次，观察页面内容变化，如果两个服务器的页面均出现图片，那么说明动静资源分离配置成功。
